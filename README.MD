# Cyber Owl Servo Module

Управление сервоприводами совы через Modbus RTU с веб-интерфейсом и REST API.

## Обзор

Проект **Cyber Owl Servo Module** — это серверное приложение на FastAPI для управления сервоприводами роботизированной совы через протокол Modbus RTU. Приложение предоставляет:

- REST API для программного управления сервоприводами
- Веб-интерфейс для ручного управления
- Поддержку предустановленных поз ("Радость", "Сон", "Готовность")
- Мониторинг состояния подключения к Modbus-устройству
- Логирование и конфигурируемые настройки

## Архитектура

Проект организован по модульному принципу:

```
app/
├── config/           # Конфигурация приложения
├── core/             # Основная логика
│   ├── httpd.py      # FastAPI-сервер
│   ├── logger.py     # Система логирования
│   ├── routers/      # API-роутеры
│   └── low_level_controllers/
│       ├── servo_controller.py        # Низкоуровневое управление сервами
│       └── servo_controller_manager.py # Менеджер контроллера
├── content/          # Статические файлы (веб-интерфейс)
└── main.py           # Точка входа
```

### Ключевые компоненты

1. **`servo_controller.py`** — потокобезопасный контроллер сервоприводов, реализующий:
   - Подключение к Modbus-устройству через `pymodbus`
   - Автоматическое переподключение при потере связи
   - Очередь команд для асинхронной обработки
   - Поддержку 4 сервоприводов:
     - `HV` (Head Vertical) — вертикальное положение головы
     - `HH` (Head Horizontal) — горизонтальное положение головы
     - `LW` (Left Wing) — левое крыло
     - `RW` (Right Wing) — правое крыло

2. **`servo_controller_manager.py`** — синглтон-менеджер, обеспечивающий централизованное управление экземпляром контроллера.

3. **`servo_router.py`** — API-роутер с эндпоинтами для управления сервами:
   - `POST /servo/happy` — радостная поза
   - `POST /servo/sleep` — поза сна
   - `POST /servo/ready` — поза готовности
   - `POST /servo/lw_sign` — махание левым крылом
   - `POST /servo/rw_sign` — махание правым крылом
   - `POST /servo/set_positions` — установка произвольных позиций
   - `GET /servo/stats` — статистика подключения

4. **`httpd.py`** — основной FastAPI-сервер с поддержкой жизненного цикла (lifespan), автоматической инициализации контроллера при старте и корректного завершения.

## Веб-интерфейс

Доступен по корневому пути `/`. Интерфейс позволяет:

- Управлять положением каждого сервопривода через слайдеры
- Выполнять предустановленные позы кнопками
- Отправлять комбинации позиций с задержкой
- Просматривать документацию API через Swagger (`/docs`) и ReDoc (`/redoc`)

Интерфейс реализован в виде статического HTML-файла `app/content/index.html`.

## Конфигурация

Настройки приложения задаются через переменные окружения (`.env` файл):

| Переменная | Описание | Обязательная | Значение по умолчанию |
|------------|---------|-------------|----------------------|
| `SERVO_PORT` | Порт сервера | Да | — |
| `SERVO_HOST` | Хост сервера | Да | — |
| `SERVO_LOG_LEVEL` | Уровень логирования | Да | — |
| `SERVO_DOC_ROOT` | Путь к статическим файлам | Нет | `./content` |
| `SERVO_MODBUS_PORT` | Порт Modbus (например, `/dev/ttyUSB0`) | Нет | `/dev/ttyUSB0` |
| `SERVO_LOGS_DIR` | Директория для логов | Нет | `None` |

Пример `.env` файла:
```
SERVO_HOST=0.0.0.0
SERVO_PORT=8001
SERVO_LOG_LEVEL=INFO
SERVO_MODBUS_PORT=/dev/ttyUSB0
SERVO_DOC_ROOT=./app/content
SERVO_LOGS_DIR=./logs
```

## Зависимости

Основные зависимости:

- `fastapi` — веб-фреймворк
- `uvicorn` — ASGI-сервер
- `pymodbus` — работа с Modbus RTU
- `python-dotenv` — загрузка переменных окружения
- `pydantic` — валидация данных

Полный список в `requirements.txt`.

## Запуск

### 1. Через Docker (рекомендуется)

Создайте `.env` файл с настройками, затем:

```bash
docker-compose up --build
```

### 2. Локально

```bash
# Установка зависимостей
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate     # Windows

pip install -r requirements.txt

# Запуск
python -m app.main
```

## Docker

Проект включает `Dockerfile` и `docker-compose.yml` для контейнеризации. Особенности:

- Используется образ `python:3.10-slim`

- Монтирование моделей и статики как read-only томов
- Автоматический запуск через `start.sh`

## API Документация

После запуска сервера документация доступна по:

- Swagger UI: `http://<host>:<port>/docs`
- ReDoc: `http://<host>:<port>/redoc`

## Безопасность и надёжность

- Контроллер работает в отдельном потоке
- Реализовано автоматическое переподключение к Modbus-устройству с экспоненциальной задержкой
- Все команды проходят валидацию диапазона значений (0–100)
- Статистика подключения доступна через API
